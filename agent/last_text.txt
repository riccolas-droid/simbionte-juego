He realizado un cambio pequeño y verificable para mejorar el juego sin romperlo:  
He añadido un límite para que el punto de fuga (vanishingPoint) no pueda salir del área visible del canvas. Esto evita que el usuario pueda arrastrar el punto fuera de la pantalla, lo que podría causar confusión o problemas de interacción.

### Cambios en backlog.js (funciones pointermove y pointerdown):

```js
function clamp(value, min, max) {
  return Math.min(Math.max(value, min), max);
}

canvas.addEventListener("pointerdown", (e) => {
  dragging = true;
  vanishingPoint = {
    x: clamp(e.clientX, 0, canvas.width),
    y: clamp(e.clientY, 0, canvas.height)
  };
});

canvas.addEventListener("pointermove", (e) => {
  if (dragging) {
    vanishingPoint = {
      x: clamp(e.clientX, 0, canvas.width),
      y: clamp(e.clientY, 0, canvas.height)
    };
  }
});
```

Así garantizamos que el punto de fuga siempre permanece visible y se asegura una mejor experiencia de usuario.
